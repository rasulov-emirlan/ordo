# Ordo — управление сетью кофеен

Контрольная башня владельца для кофеен и кафе Кыргызстана и Центральной Азии:
персонал и смены, удержания (депремирование) по инцидентам, камеры с
видеоаналитикой, подсчёт посетителей, продажи и аналитика по всем точкам.

**Демо:** https://ordo.night.enkiduck.com — логин `demo` / `demo2026`.

## Что внутри

- **Обзор сети** — выручка/гости/конверсия по всем точкам, очередь инцидентов.
- **Персонал** — карточки сотрудников, ставки (оклад / за смену / почасовая),
  журнал начислений, расчёт зарплаты за период.
- **Смены** — недельный график по точкам, статусы (отработана/прогул/больничный).
- **Инциденты** — события от видеоаналитики и ручная фиксация; утверждение
  менеджером превращает событие в удержание в журнале зарплаты. Оформление —
  как депремирование (денежные штрафы не являются дисциплинарным взысканием по
  ТК КР в редакции 2025).
- **Камеры** — HLS-потоки в приложении, зоны (вход/касса/зал/кухня), вебхук для
  внешней видеоаналитики, счётчик посетителей.
- **Продажи** — киоск-касса (без фискализации — Ordo не ККМ) и журнал чеков;
  способы оплаты: наличные / карта / QR (ELQR).
- **Статистика** — динамика выручки и трафика, пиковые часы, топ товаров,
  дисциплина, сравнение точек.

Рынок и позиционирование: `docs/market-research.md` (Poster/iiko, фискализация
ГНС, ELQR, Hikvision/ISAPI, правовой статус штрафов).

## Стек

Next.js 15 (App Router, RSC) + better-sqlite3, без внешних сервисов. Вся БД —
один файл SQLite; первый запуск на пустой базе сам наполняет демо-данными.

## Разработка

```bash
pnpm install
pnpm dev          # http://localhost:3000, БД в ./data/ordo.db
pnpm test         # vitest: payroll / incidents / stats
pnpm typecheck
```

## Деплой

```bash
docker compose up -d --build   # Traefik → https://ordo.night.enkiduck.com
```

## Интеграции

- `POST /api/integrations/detections` — событие видеоаналитики → инцидент в
  очередь (Bearer API-ключ из Настроек).
- `POST /api/integrations/visits` — счётчик посетителей (камера входа,
  Hikvision ISAPI people counting, ИК-датчик).
- Планы: импорт продаж из Poster POS API, iiko Cloud API; RTSP→HLS через
  go2rtc/MediaMTX на площадке.
