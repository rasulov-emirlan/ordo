# Спека A — Персонал и смены

Сначала прочитай `docs/CODEX-GUIDE.md` (правила, библиотеки, CSS-классы) и
референс `src/app/(app)/page.tsx`. Работай ТОЛЬКО в:

- `src/app/(app)/staff/**`
- `src/app/(app)/roster/**`

## 1. `/staff` — список персонала

- Фильтр по заведению через query `?venue=<id>` — ряд `<Link>`-кнопок
  (`btn btn--sm btn--ghost`, активная без `--ghost`) + «Все точки».
- Таблица: Имя (ссылка на `/staff/[id]`), Должность, Точка, Тип оплаты
  (по-русски: «оклад» / «за смену» / «почасовая»), Ставка (`som`), Телефон,
  Статус (badge; «неактивен» — muted). Показывать и неактивных
  (`listEmployees(venueId, true)`), неактивных в конце.
- Карточка «Добавить сотрудника»: имя, телефон, заведение (select), должность,
  тип оплаты (select), ставка (number). Server action → INSERT, revalidate.

## 2. `/staff/[id]` — карточка сотрудника

- Шапка: имя, должность, точка, телефон, дата найма (`fmtDate`), статус +
  кнопка «Деактивировать»/«Вернуть в штат» (server action, UPDATE status).
- **Зарплата за текущий месяц** (с 1-го числа по сегодня): собрать смены
  (`SELECT * FROM shifts WHERE employee_id=? AND date BETWEEN ...`) и
  начисления (`SELECT kind, amount FROM adjustments WHERE employee_id=? AND date BETWEEN ...`),
  вызвать `computePayroll(...)` из `@/lib/payroll` и показать stat-grid:
  Начислено (base), Премии, Удержания (красным), Авансы, **К выплате** (total,
  `card--framed`-подача или `stat__value--red` если < 0), Смен отработано.
- **Журнал начислений**: таблица adjustments (дата, тип по-русски: «удержание
  (штраф)» / «премия» / «аванс», сумма ±, причина). Тип «удержание» — сумма
  красным с минусом.
- Форма «Добавить запись»: тип (select: премия/удержание (штраф)/аванс), сумма,
  причина, дата (default сегодня). Server action → INSERT в adjustments
  (venue_id сотрудника, created_by = user.id).
  Под формой строка `small muted`: «По ТК КР денежные штрафы не являются
  дисциплинарным взысканием — оформляйте как депремирование по локальному акту.»
- **Смены за 30 дней**: компактная таблица (дата, время, статус по-русски:
  запланирована/отработана/прогул/больничный; прогул — badge--red).

## 3. `/roster` — график смен на неделю

- Query: `?venue=<id>` (default — первое заведение) и `?week=<YYYY-MM-DD>`
  (понедельник; default `weekStart(todayISO())`). Навигация: «← Пред»,
  «Текущая», «След →» ссылками; переключатель заведений как на /staff.
- Сетка: строки — активные сотрудники точки, колонки — 7 дней (`WEEKDAYS_RU` +
  число). Ячейка: если смена есть — время `HH:MM–HH:MM` (mono, small) +
  статус-маркер (отработана — обычная, прогул — badge--red «прогул», больничный
  — muted «болел», запланирована — ghost); если нет — тонкая кнопка «+».
- Добавление смены: ячейки-«+» ведут на `?venue&week&add=<empId>:<date>` —
  под сеткой рендерится карточка-форма (время с/до, default часы работы точки)
  → server action INSERT.
- Клик по существующей смене (`?edit=<shiftId>`) — карточка под сеткой: сменить
  статус (4 кнопки) или «Удалить смену». Server actions, revalidate `/roster`.
- Итог справа в шапке: смен на неделе, из них отработано.

## Критерии готовности

`pnpm typecheck` проходит. НЕ запускай `pnpm build`/`dev`/`install`. Русский UI,
существующие CSS-классы, server actions с `requireUser()`.
