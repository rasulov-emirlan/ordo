# Спека C — Продажи, статистика, заведения, настройки

Сначала прочитай `docs/CODEX-GUIDE.md` (правила, библиотеки, CSS-классы) и
референс `src/app/(app)/page.tsx`. Работай ТОЛЬКО в:

- `src/app/(app)/sales/**`
- `src/app/(app)/stats/**`
- `src/app/(app)/venues/**`
- `src/app/(app)/settings/**`

## 1. `/sales` — касса и журнал продаж

- Переключатель заведения `?venue=<id>` (default первое) — btn--sm ссылки.
- **Касса** (client component `"use client"`, корзина в useState):
  сетка товаров по категориям (`listProducts()`, кнопки-карточки: название +
  цена mono); клик добавляет в корзину. Корзина справа (grid-2): строки
  (название × qty, сумма, кнопка −), итог крупно (mono), способ оплаты — 3
  кнопки-переключателя: «Наличные», «Карта», «QR (ELQR)», продавец (select
  активных сотрудников точки), кнопка «Пробить чек» → server action:
  INSERT sales + sale_items транзакцией (цены берёт из БД по product_id, НЕ
  доверяет клиенту), `ts: nowISO()` → revalidate. После успеха корзина
  очищается (форма через action + redirect с `?ok=1`, или useTransition —
  на твой вкус, но без новых зависимостей).
- Пометка `small muted` под кассой: «Ordo не является ККМ: фискализация чеков —
  в вашей кассе (О!Касса, eKassa) или через POS-интеграцию (Poster).»
- **Журнал за сегодня**: таблица последних 30 чеков (время, продавец, позиции
  строкой «Латте ×2, Круассан», способ оплаты по-русски, сумма) + stat-строка:
  выручка/чеков/средний чек за сегодня по точке (`periodSummary`).

## 2. `/stats` — аналитика

- Период `?days=7|30|90` (default 30) + `?venue=<id|all>` — ряды btn--sm ссылок.
- stat-grid: Выручка, Чеков, Средний чек, Посетители, **Конверсия в покупку**
  (%), Удержания за период (`finesTotal`, красным).
- `card`: Выручка по дням — `<Bars>`; `card`: Посетители по дням — `<Bars>`.
- grid-2:
  - «Пиковые часы» — `visitorsByHour`, `<Bars>` с label «08:00»…; подпись
    muted «суммарно посетителей за период по часам суток».
  - «Способы оплаты» — таблица метод (по-русски: Наличные/Карта/QR (ELQR)) /
    сумма / доля % (mono, самая большая доля красным).
- grid-2:
  - «Топ товаров» — `topProducts`, таблица: товар, категория, шт, выручка.
  - «Дисциплина» — `disciplineBoard(from, to)` топ-8: сотрудник (ссылка на
    `/staff/[id]`), точка, удержания (красным если > 0), нарушений, прогулов.
- «Сравнение точек» — таблица по каждому заведению за период: выручка, чеков,
  средний чек, посетители, конверсия %, выручка/посетитель (mono). Лучшую
  выручку выделить красным числом.

## 3. `/venues` и `/venues/[id]`

- `/venues`: карточки заведений (grid-3): название (ссылка), адрес/город,
  часы работы mono, сегодня: выручка + гости; кнопка-карточка «Добавить
  заведение» → форма (название, адрес, город, часы) server action INSERT.
- `/venues/[id]`: шапка (название, адрес, часы), stat-grid за сегодня
  (`periodSummary`): выручка, чеков, средний чек, гости, конверсия.
  - «Сейчас на смене»: сотрудники со сменой на сегодня (JOIN shifts,
    date=today): имя (ссылка), должность, время смены; пусто — «Сегодня смен
    нет».
  - «Камеры» — мини cam-grid этой точки (HlsPlayer, ссылки на /cameras/[id]).
  - «Последние инциденты» — `listIncidents({venueId, limit: 8})` таблицей.
  - `<Bars>` выручка 14 дней.
  - Форма-карточка «Редактировать»: название/адрес/город/часы → UPDATE.

## 4. `/settings`

- «Профиль»: имя, логин, роль (по-русски «владелец»/«менеджер») + кнопка
  «Выйти» → импортируй `logoutAction` из `@/app/login/actions`.
- «Пользователи»: таблица users (имя, логин, роль, точка или «все»); форма
  добавления (имя, логин, пароль, роль select, точка select опционально для
  менеджера) → INSERT с `hashPassword()` из `@/lib/auth`.
- «Типы нарушений»: таблица `listIncidentTypes()`: название, код mono, сумма
  удержания по умолчанию (инлайн-форма: input + btn--sm «Сохранить» на строку,
  server action UPDATE default_fine), severity badge, «детектируется» ●/—.
- «API-ключи»: таблица api_keys (название, токен mono, точка или «все», дата);
  форма создания (название, точка опц.) — токен генерируй
  `"ordo_" + crypto.randomBytes(24).toString("base64url")` (`node:crypto`).
  Кнопка «Отозвать» (DELETE).
- `card` «Интеграции» — статичный текст (по 1-2 строки на пункт):
  - Вебхуки Ordo: `POST /api/integrations/detections` (события видеоаналитики),
    `POST /api/integrations/visits` (счётчик посетителей), auth Bearer API-ключ.
  - Poster POS: импорт продаж по API — в разработке; iiko Cloud API — план.
  - Камеры: RTSP (Hikvision/Dahua) → HLS через go2rtc/MediaMTX; Hikvision
    ISAPI people counting → вебхук visits.
  - Фискализация: Ordo не ККМ; чеки — О!Касса/eKassa/через POS.
- `card` «Правовая справка (КР)»: 3-4 строки — денежные штрафы не являются
  дисциплинарным взысканием по ТК КР (ред. 2025); Ordo оформляет удержания как
  депремирование: локальный акт о премировании, приказ по каждому случаю,
  ознакомление сотрудника; материальная ответственность (недостача) — отдельно,
  с документированным ущербом.

## Критерии готовности

`pnpm typecheck` проходит. НЕ запускай `pnpm build`/`dev`/`install`. Русский UI,
существующие CSS-классы, server actions с `requireUser()`.
